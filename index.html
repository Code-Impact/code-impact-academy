What this is
(and what it honestly isn't)

This isn't a "type after me" tutorial. The course gives you objectives and the questions to answer; you design the spec and the architecture, and AI agents help you build it. You learn to think like an engineer who directs the work — not just one who writes lines.

  • ✓AI writes the code; you review it and can explain every part.
  • ✓It really deploys — Vercel and Supabase online, your container on AWS.

The one rule

You may let AI write the code — but you may not ship what you can't explain. If you can't describe what a file does, that's the next thing you learn, not the next thing you merge.

Where it runs

Local first, cloud last

You'll have a working, online app well before you touch AWS. The front end and data go online early; the backend starts in a local container and moves to the cloud as the final step — so the scary part comes when you're ready for it.

ONLINE · Wk 4

Front end

→ Vercel

Your React app is deployed and shareable from week 4, and auto-deploys from GitHub later.

ONLINE · Wk 5–6

Data & auth

→ Supabase

Gmail login and your database live in Supabase's cloud from the start — no server to manage.

LOCAL · Wk 8

Backend

→ Docker (your machine)

The container runs on your laptop first, talking to the online Supabase. Prove it works before the cloud.

🚀

Final step — Wk 11: the same Docker image moves to AWS (via ECR + App Runner/ECS/EC2). Nothing about your app changes — that's the payoff of containers: build once, run anywhere.

The stack

Tools you'll actually use

Deliberately small. Each tool has a clear job and a free tier to start. Tap a card to open its site.

Antigravity

Wk 1+

Google's agent-first IDE — a VS Code fork where AI agents plan, write and verify code. Free preview, Gmail sign-in. This is where you build. →

Supabase

Wk 5+

Postgres database plus ready-made Google login. Your data and auth, hosted, with access rules you control. →

Vercel

Wk 4+

One-command hosting for the front end, with instant previews and auto-deploys from GitHub. →

GitHub Actions

Wk 10+

Your CI/CD pipeline. On every push it builds, checks, and deploys — the engine of GitOps. →

Docker

Wk 8+

Package the backend so it runs the same on your laptop and in the cloud. Build once, run anywhere. →

AWS

Wk 11

The cloud your container ships to at the end: a registry (ECR) plus a runtime (App Runner/ECS/EC2). →
💳

Cost & safety: everything here has a free tier, but AWS can bill real money if left running. You'll set a budget alert on day one of the AWS week and tear everything down in week 12. Never commit secrets — use environment variables from the start.

Get set up

Before Week 1

Set up your cockpit and accounts first, so week 1 is about building, not installing.

✅ Your checklist

  • ✓ A laptop (Windows, Mac or Linux) and a personal Gmail account.
  • ✓ Install Antigravity from antigravity.google — it's a full IDE (a VS Code fork), not a plugin. Import your VS Code settings if you have them.
  • ✓ A GitHub account, with Git installed and working.
  • ✓ Free accounts ready: Supabase, Vercel, and a Google Cloud project for the login keys.
  • ✓ Docker Desktop installed (you'll use it from week 8).
  • ✓ An AWS account for the final week — with a billing budget alert set.
